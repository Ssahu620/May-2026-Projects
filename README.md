# May-2026-Projects

## Conversion Rate & Rebate Analysis

Identifies items where Amazon conversion rates increased month-over-month **without** any Deal or Coupon Rebate active that month, then further classifies those organic increases by whether retail price fell — distinguishing truly organic growth from price-cut-driven lift.

---

## Files

| File | Description |
|---|---|
| `v2.Conversion Rate.xlsx` | Monthly conversion rates per item/ASIN (Jan–May 2026 and Jan–May 2025) |
| `v2.2026 VC Master Data.xlsx` | Weekly VC data including Deals Rebate and Coupons Rebate columns |
| `v2.Retail Prices.xlsx` | Monthly retail prices per item/ASIN (Dec 2025–Apr 2026; May 2026 absent) |
| `cr_rebate_analysis.py` | Python script that runs the analysis and produces the output file |
| `CR_Increase_No_Rebate_Analysis.xlsx` | Formatted output workbook with 6 sheets (generated by the script) |

---

## How to Run

```
python cr_rebate_analysis.py
```

Requires Python 3 with `pandas` and `openpyxl` installed:

```
pip install pandas openpyxl
```

> Close `CR_Increase_No_Rebate_Analysis.xlsx` in Excel before running, or the script will raise a clear error asking you to do so.

## Algorithm

### Step 1 — Load data
- Read `v2.Conversion Rate.xlsx` — 345 items, monthly CR columns for 2025 and 2026
- Read `v2.2026 VC Master Data.xlsx` — 91,476 weekly rows with Deals Rebate and Coupons Rebate fields
- Read `v2.Retail Prices.xlsx` — monthly retail prices per item (Dec 2025–Apr 2026)

### Step 2 — Find conversion rate increases
For each item, compare consecutive month pairs within 2026 in chronological order:

```
Jan 2026 → Feb 2026
Feb 2026 → Mar 2026
Mar 2026 → Apr 2026
Apr 2026 → May 2026
```

An **increase** is recorded when:
- Both months have a non-null conversion rate, **and**
- The current month's rate is strictly greater than the previous month's rate

Each qualifying instance is one row containing: Item, ASIN, Brand, Month, Prev Month, Prev CR, Curr CR, and **CR Change** (absolute pp difference = Curr CR − Prev CR).

### Step 3 — Find months with rebates
From the VC Master Data file, extract every `(Item, MonthYear)` combination where either:
- `Deals Rebate > 0`, **or**
- `Coupons Rebate > 0`

Duplicate item-month pairs are collapsed to one record per item per month.

### Step 4 — Split by rebate presence
Left-join the CR-increase records to the rebate months on `(Item, Month)`:
- **No match** → CR increased with no rebate active → `no_rebate`
- **Match** → CR increased but a rebate was active → `with_rebate`

Item columns are cast to `float` before the merge to prevent type-mismatch misses.

### Step 5 — Classify no-rebate items by retail price change
For each row in `no_rebate`, look up the retail price in both the current and previous month from the Retail Prices file. Each row receives a `Price Direction` tag:

| Condition | Price Direction | Goes to |
|---|---|---|
| `curr_price < prev_price` | Down | `price_down` (Red sheet) |
| `curr_price > prev_price` | Up | `price_stable` (Lime sheet) |
| `curr_price == prev_price` or no price data | Flat / No Data | `price_stable` (Lime sheet) |

Adds `Prev Price`, `Curr Price`, `Price Change`, and `Price Direction` columns to both subsets. Rows where May 2026 price is unavailable (file stops at Apr 2026) get `"Flat / No Data"` and land in `price_stable`.

### Step 6 — Build supporting tables
- **Rebate Detail**: aggregates total Deals and Coupons Rebate dollars per item/month for all months where either rebate > 0
- **Brand Summary**: built from `price_stable` — items where CR increased, no rebate was active, and retail price did **not decrease**. This is the same dataset as the D5 "Unique Items" KPI box, so the two numbers always match.

### Step 7 — Write formatted output
Saves `CR_Increase_No_Rebate_Analysis.xlsx` with professional management-presentation formatting:
- Navy/blue header color scheme, alternating row shading, freeze panes, auto column widths
- CR columns formatted as `0.00%`, rebate/price columns as `$#,##0.00`
- Color-coded sheet tabs matched to KPI boxes in the Executive Summary

| Sheet | Tab Color | Matching KPI Box | Contents |
|---|---|---|---|
| `Executive Summary` | Navy `1F3864` | — | 6 KPI boxes + brand table + methodology note |
| `CR Up - No Rebate` | Green `538135` | CR Increased No Rebate | All no-rebate CR increases (parent list before price split) |
| `No Rebate - Price Down` | Red `C00000` | No Rebate — Price Decreased | No-rebate items where retail price fell that month |
| `No Rebate - Price Stable` | Lime `70AD47` | Unique Items / Price Stable | No-rebate items where price held or rose |
| `CR Up - Had Rebate` | Orange `C55A11` | CR Increased With Rebate | CR increases where a rebate was active that month |
| `Rebate Detail` | Blue `2E75B6` | Total CR Increase Instances | All item-months with non-zero rebate, with dollar totals |

## Executive Summary KPI Definitions

| KPI Box | Source data | Definition |
|---|---|---|
| Total CR Increase Instances | `cr_increases` | All item-month pairs where CR rose vs. prior month |
| CR Increased With Rebate | `with_rebate` | Instances where a Deal or Coupon Rebate was active that month |
| CR Increased No Rebate | `no_rebate` | Instances with no Deal or Coupon Rebate active |
| **Unique Items (No Rebate, No Price Decrease)** | **`price_stable`** | **Distinct items where CR rose, no rebate was active, and retail price did NOT fall** |
| No Rebate — Price Decreased | `price_down` | Instances where no rebate was active but retail price fell |
| No Rebate — Price Stable / Up | `price_stable` | Instances where no rebate was active and price held or rose |

**Consistency rule:** The D5 "Unique Items" KPI box and the Brand Summary table both source from `price_stable`, so cell D5 and the brand summary TOTAL row (column C) always show the same unique item count and instance total respectively.

**Additive check:** E5 (`price_down`) + F5 (`price_stable`) = total `no_rebate` instances (63 + 223 = 286 ✓).

## Adding Monthly Data Through 2026

**No code changes needed as you add months.** `MONTHS` already contains all 12 months of 2026. Each time a new month's data is ready:

1. Add the new month column to `v2.Conversion Rate.xlsx` (e.g., `Jun 2026`)
2. Add the same month column to `v2.Retail Prices.xlsx`
3. Ensure the new month's weekly rows are in `v2.2026 VC Master Data.xlsx`
4. Run `python cr_rebate_analysis.py`

The script automatically detects which month columns are present and skips the rest. At startup it prints:

```
Months found in CR file: Jan 2026, Feb 2026, ..., Jun 2026
```

The Executive Summary methodology note updates automatically to reflect the actual months analyzed.

## Multi-Year Support

### What works automatically (no code changes)

| Function | Why it is year-agnostic |
|---|---|
| `find_rebate_months` | Reads `MonthYear` column — accepts any year present in the VC file |
| `split_by_rebate` | Joins on `(Item, Month)` string — works for any month label |
| `classify_by_price` | Looks up price columns by name — works for any year in the Retail Prices file |
| All formatting / output functions | Driven entirely by the data passed in |

### What requires changes for additional years

**1. Update `MONTHS` in the script** — extend with the new year's months in strict chronological order:

```python
MONTHS = [
    "Jan 2026", "Feb 2026", ..., "Dec 2026",
    "Jan 2027", "Feb 2027", ..., "Dec 2027",   # add when needed
]
```

**2. Merge VC Master Data files** — `INPUT_VC` is hardcoded as `"v2.2026 VC Master Data.xlsx"`. If another year's rebate data is in a separate file, those CR increases are silently treated as "no rebate." Combine all years into one file, or load and `pd.concat()` multiple files in `load_data()`.

**3. Expand the Retail Prices file** — months absent from the file default to `"Flat / No Data"` (safe, no evidence of a price drop).

### Critical risk — the gap problem

Only extend `MONTHS` across a year boundary once you have complete 12-month data for the prior year in the CR file. Partial data creates false comparisons:

```
Dec 2026  →  Jan 2027   ← OK if both columns exist ✓
May 2026  →  Jan 2027   ← 8-month gap if Jun–Dec 2026 absent ✗
```
