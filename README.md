# May-2026-Projects

## Conversion Rate & Rebate Analysis

Identifies items where Amazon conversion rates increased month-over-month **without** any Deal or Coupon Rebate active that month. This helps distinguish organic conversion improvements from promotion-driven ones.

---

## Files

| File | Description |
|---|---|
| `v2.Conversion Rate.xlsx` | Monthly conversion rates per item/ASIN (Jan–May 2026) |
| `v2.2026 VC Master Data.xlsx` | Weekly VC data including Deals Rebate and Coupons Rebate columns |
| `cr_rebate_analysis.py` | Python script that runs the analysis and produces the output file |
| `CR_Increase_No_Rebate_Analysis.xlsx` | Output file with 4 analysis sheets (generated by the script) |

---

## How to Run

```
python cr_rebate_analysis.py
```

Requires Python 3 with `pandas` and `openpyxl` installed:

```
pip install pandas openpyxl
```

> Close `CR_Increase_No_Rebate_Analysis.xlsx` in Excel before running, or the script will raise a clear error asking you to do so.

## Algorithm

### Step 1 — Load data
- Read `v2.Conversion Rate.xlsx` (345 items, monthly CR columns for 2025 and 2026)
- Read `v2.2026 VC Master Data.xlsx` (91,476 weekly rows with rebate fields)

### Step 2 — Find conversion rate increases
For each item, compare consecutive month pairs within 2026 in chronological order:

```
Jan 2026 → Feb 2026
Feb 2026 → Mar 2026
Mar 2026 → Apr 2026
Apr 2026 → May 2026
```

An **increase** is recorded when:
- Both the current and previous month have a non-null conversion rate, **and**
- The current month's rate is strictly greater than the previous month's rate

Each qualifying instance is recorded as one row: Item, ASIN, Brand, Month, Prev Month, Prev CR, Curr CR.

### Step 3 — Find months with rebates
From the VC Master Data file, extract every `(Item, MonthYear)` combination where either:
- `Deals Rebate > 0`, **or**
- `Coupons Rebate > 0`

Duplicate item-month pairs are collapsed to one record per item per month.

### Step 4 — Split by rebate presence
Left-join the CR-increase records to the rebate months on `(Item, Month)`:
- **No match** → CR increased with no rebate active → goes into the primary output sheet
- **Match** → CR increased but a rebate was active → goes into the reference sheet

### Step 5 — Build supporting tables
- **Rebate Detail**: aggregates total Deals and Coupons Rebate dollars per item/month for all months where either rebate > 0
- **Brand Summary**: counts unique items and total CR-increase instances per brand, excluding subtotal rows

### Step 6 — Write output
Saves `CR_Increase_No_Rebate_Analysis.xlsx` with four sheets:

| Sheet | Contents |
|---|---|
| `CR Up - No Rebate` | Primary list — items where CR rose with no rebate that month |
| `CR Up - Had Rebate` | Reference — items where CR rose but a rebate was active |
| `Rebate Detail` | All item-month combos with non-zero Deal or Coupon Rebate, with totals |
| `Summary by Brand` | Count of qualifying items and instances per brand |
